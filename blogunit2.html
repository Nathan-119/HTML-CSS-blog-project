1.5

Describing copyright and other constraints affecting websites

When it comes to copyright, it’s generally best practice to assume its under copyright, therefore it’s best to explore creation of your own images. When a copyright claim is made, it’s considered standard to remove the infringing material as soon as possible, as seen under the Digital Millennium Copyright Act (DMCA). [1]

When a claim complaint is made, it should be responded to, to assure the potential copyright holder that the claim is being investigated. Finally to ensure you are able to use copyrighted material, ensure you have written permission from the copyright holder, or otherwise investigate that it is under fair use. [2]

1.6

Describing access issues that need taking into account

When designing a website it's generally a good idea to consider users who are shortsighted or hard of seeing, or various states of colourblindness. The best way to tackle these would be through larger font options, and colourblind options.

1.7

Describe which are the appropriate file types for websites>

To begin with, I'll explore the basic file types necessary to run a webpage. [3] .Html is your primary filetype for browsing the internet, standing for hyper text markup language, a .html file will instruct your browser on how to display the contents of an html document using elements. The latest version of HTML is HTML5, produced in 2014. [4]
.css is a file that stands for cascading style sheet, when creating pages with .html the style of the page had to be defined within each html document, which would often be a painful process. .css allows for the style of the page to be defined in one location and then the .html file will pull the style information for each web page from a singular location. [5]

In addition to these two basic files, there’s also other files that you may choose to put into your website, such as image files, or audio.
In fact, most file types are appropriate for websites, any browser that was made in the last 5 years can handle a variety of file types, so it comes down to preference at that point.

To begin with images, png is objectively better than jpegs for images, the artifacting present in repeated opening of jpegs and the overall quality loss for compression reasons, but in this day and age, image files do not need to be compressed unless you live in the USA where you live with download limits, and even then the size difference between png and jpeg is so negligible that you wouldn’t consider it to be an issue .Gifs are also a valid file type to play a short animation.

Audio file types are generally all the same to the average user, although .flac is to be avoided for websites as the benefit isn’t really worth it for something only audiophiles will appreciate, best to stick with .mp3 files

.webm files have started to gain popularity for displaying videos as the quality of the video is superior and are strong competition for the current industry standard of .mp4, however support for .webm is coming in somewhat slowly and some browsers and iphones may not run a .webm, however taking into account the market share of iphones is lesser than android, which does support .webm, you may find that an acceptable compromise. [6]

.pdfs can be run in browser as modern browsers have built in javascript to display .pdfs In summary, it’s safe to assume that most file types are appropriate unless they’re for a highly specialised non-consumer software.
